§1. PHASE 1 — RESEARCH & BRAND INTELLIGENCE (if client site exists)
═══════════════════════════════════════════════════════════════════════════════

CONDITIONAL: If the client already has a website, execute this phase FIRST.
If building from scratch with no existing site, skip to §2.

USING FIRECRAWL OR WEB SCRAPING:

Step 1 — Client Brand Extraction:
  □ Scrape client's current site
  □ Extract: Logo URLs, brand colors (from CSS vars + inline styles),
    font families, tone of voice, key messaging, existing content,
    full URL architecture via sitemap/map
  □ Save as: research/01-client-brand.md
  □ Include a "Brand Snapshot" summary block:
    Company | Primary Color (hex) | Secondary Color (hex) |
    Accent Color (hex) | Fonts | Tone | Core Message

Step 2 — Competitive Niche Analysis:
  □ Find top 10 competitors in the same niche
  □ Score each on 8 dimensions (1-10 scale):
    Search Visibility | Review Quality | Visual Design | Mobile UX |
    Content Depth | Social Proof | CTA Strategy | Page Speed
  □ Deep scrape top 5: visual identity, content strategy,
    site architecture, conversion strategy
  □ Identify 3-5 "Top 10% Patterns" — what elite stores do that
    average ones don't
  □ Save as: research/02-competitor-analysis.md

Step 3 — Build Brief:
  □ Merge brand extraction + competitor analysis into master brief
  □ Include: Design Direction (palette, typography, animations, what to AVOID),
    Site Architecture (pages, nav, content hierarchy, CTA strategy),
    Content Framework (3 headline options, SEO keywords, section copy direction),
    Conversion Playbook (primary goal, lead capture, social proof plan, trust signals)
  □ Save as: research/03-build-brief.md
  □ ALSO generate client-facing report: research/niche-analysis-report.md
    (Industry benchmarks, competitor table, rating averages, key opportunities,
    recommended approach — formatted as a professional lead magnet)

IMPORTANT RULES FOR THIS PHASE:
  • Always scrape the existing site first. Never start from scratch
    when brand assets exist online.
  • Be opinionated. Pick specific hex colors, specific fonts, specific
    animations. Justify each choice with competitor data.
  • The niche report is a SALES TOOL. Format it as something impressive
    enough to email to a cold prospect.

─── COSMIC BACKGROUND SYSTEM ───

    diamond particles (representing card backs) drifting upward at

    NOT an external file)
  • Faint scanline overlay at 2% opacity (CRT video game nod)

    fade-out at 70% from center

§5. ANIMATION SPECIFICATION (GSAP + Framer Motion + CSS)
═══════════════════════════════════════════════════════════════════════════════

GSAP (global/complex timelines):
  • Hero H1:     word-by-word stagger (y:60→0, opacity:0→1, skewY:4→0,
                 duration:0.75, stagger:0.12, ease:"power3.out")
  • Stats:       ScrollTrigger count-up, start:"top 80%", 2s, "power2.out"
  • Event slots: ScrollTrigger width animation 0→%
  • Buylist:     ScrollTrigger scrub:1, translateX(±60px)→0
  • Category cards: stagger {amount:0.6, from:"start"}, y:50→0, scale:0.96→1

Framer Motion (component-level):
  • Sections:    initial:{y:50, opacity:0} whileInView:{y:0, opacity:1}
                 transition:{duration:0.6} viewport:{once:true, margin:"-100px"}
  • Products:    staggerChildren 0.08, hover scale(1.03) + rarity glow
  • Categories:  staggerChildren 0.06
  • Story avatars: staggerChildren 0.1
  • Modal:       scale(0.8→1), opacity 0→1 on open
  • 3D card tilt: useMotionValue(x/y) → rotateX/rotateY at ±15deg

CSS Animations:
  • Cosmic particles:  keyframe drift y:0→-100vh, randomized x-drift
  • Shimmer button:    keyframe sweep translateX:-100%→100%, 0.6s on hover
  • Blinking dot:      keyframe pulse opacity 1→0.3→1, 1.5s infinite
  • Card foil shimmer: linear-gradient bg-position animation (holo cards)
  • Countdown digits:  flip card animation

CRITICAL ANIMATION RULES (from audit):
  • Only animate transform and opacity (GPU-composited properties)
  • will-change: transform, opacity on continuously animated elements (F3.5, F7.3)
  • All UI interaction transitions ≤ 300ms (F2.5, F7.1)
  • Use back.out(1.7) instead of elastic.out for spring-like entries (F7.2)
  • Stagger range: 30-100ms for lists (verified in audit)
  • @media (prefers-reduced-motion: reduce) → disable ALL animations,
    keep transitions max 0.2s. GSAP must check window.matchMedia() (F1, F7)

§10. IMMUTABLE RULES
═══════════════════════════════════════════════════════════════════════════════

1. RESEARCH FIRST. If the client has an existing site, always scrape
   and analyze before designing. Never start from scratch when brand
   assets exist online.

2. SAVE DELIVERABLES. Each research file is a tangible deliverable
   the user can share with the client or use as a sales asset.

3. BE OPINIONATED. Pick specific hex colors, specific fonts, specific
   easing curves. Justify each choice with competitor data or UX research.

4. ACCESSIBILITY IS NON-NEGOTIABLE. Every item in §6 must pass.
   No "we'll fix it later" — build accessible from the start.

5. CONDITIONAL SECTIONS. Respect siteConfig.features toggles.
   Only render sections the store actually needs.

6. THE NICHE REPORT IS A SALES TOOL. Format it as something impressive
   enough to email to a cold prospect and close the deal.

7. PERFORMANCE = MONEY. Every 100ms of load time costs conversions.
   Target Lighthouse 90+ on all metrics.

8. MOBILE-FIRST. 68% of traffic in the TCG/anime niche is mobile.
   Design for 375px first, scale up.

9. SPEED OF DELIVERY. The whole process should feel fast and automated,
   not like a consulting engagement. Ship, don't deliberate.

10. LEAVE CLEAR PLACEHOLDERS. 3D asset slots, API endpoints, and
    credential fields must be clearly marked with comments so the
    client can drop in their assets without reading the full codebase.
